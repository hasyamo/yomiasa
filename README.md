# YOMIASA

好きなnoteクリエイターの記事を、年月でたどる小さな道具です。

note標準UIでは過去記事を「もっと見る」で何度も読み込む必要があり、記事を開いて一覧へ戻ると最初から読み直しになります。YOMIASA は特定クリエイターの記事を年月別に整理し、集中して漁れるようにします。

## 特徴

- noteのURL または ID でクリエイターを登録（プロフィールをプレビュー表示）
- 複数クリエイターを登録、選択した1人の記事に集中
- 記事を年月別にグルーピング表示（サムネイル付き）
- タイトル検索 / 年・月フィルタ / 未読のみ表示
- 初回取得時の「初期既読セットアップ」で、過去に読んだ年月をまとめて既読化
- 記事ごとの既読トグル、記事を開いて戻ると読了確認
- クリエイターカード・記事一覧ヘッダーに既読率の進捗バー
- 認証・サーバー不要。状態はすべてブラウザの localStorage に保存

## 構成

ビルド不要の静的Webアプリです。

```
index.html
style.css
app.js
```

そのまま GitHub Pages / Cloudflare Pages 等に配置して動きます。

## ローカルで開く

任意の静的サーバで配信してください。例:

```sh
python3 -m http.server 4173
# http://localhost:4173
```

## 記事取得について

note.com の API はブラウザから直接呼ぶと CORS で弾かれるため、CORS 対応の中継プロキシ経由で取得しています（`app.js` 冒頭の `PROXY_URL`）。

## 技術メモ

- フレームワーク・外部ライブラリなし（Vanilla JS）
- 状態は単一の `state` オブジェクトに集約し localStorage（キー `yomiasa:v0`）へ保存
- 画面はハッシュルーティングの2画面構成（`#list` クリエイター一覧 / `#read` 記事一覧）
