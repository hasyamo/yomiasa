## 全体構成（二部構成）

### 第1部：覚醒前（クイズ修行編）
- 対象：ワイ語ほぼゼロの黎明期記事（2025年9月以前、約58記事）
- ✨マークが光る記事をタップ → 4択クイズに正解 → 終焉の鍵を入手
- 鍵3本を消費してボスに挑戦 → 撃破でランクアップ
- REAPER（E級）→ ARMORED WARRIOR（C級）→ WING OF DEATH（A級）→ **S級覚醒**

### 覚醒の山場
- WING OF DEATH（A級ボス）撃破 = S級覚醒
- 実データ上、ワイ語が爆発したのは2025年10月 = この演出に対応
- 金縁エフェクト＋システムメッセージ演出

### 第2部：覚醒後（ワイ収集編）
- 対象：覚醒後の全記事（クイズなし）
- 記事をタップ → バックグラウンドで本文取得 → ワイ語（/ワイ/g）をカウント
- 記事行にワイ語チップが出現 → タップで累計（totalWai）に加算
- totalWaiが閾値に達すると覚醒後ボスが出現 → 挑戦ボタンで撃破

### ワイ語チップの3状態

| 状態 | 表示 | タップ |
|---|---|---|
| 計測済み・未回収・ワイ数>0 | 「ワイ 7」 | 可（totalWaiに加算） |
| 計測済み・ワイ数=0 | 「ワイ 0」 | 不可（黎明期記事がゼロで埋まるのも可視化） |
| 回収済み | 「ワイ 7 ✓」 | 不可（二重取り防止） |
| 未計測（記事未タップ） | 表示なし | — |

- 収集（カウント）と回収（totalWai加算）は分離している
- 記事をタップした瞬間にバックグラウンドで本文取得・カウントが走る（遷移をブロックしない）
- カウント完了後にチップが出現する（タップ直後は出ないことがある）

## 発動フロー

1. KITAcoreさんのクリエーターカードを追加・記事取得
2. アバター画像を**ダブルタップ** → プレイヤーID入力モーダル
3. 自分のnote IDを入力 → 「認証」でプロフィール取得・プレビュー表示 → 「決定」でモード発動
4. 2回目以降はダブルタップで即ON（認証スキップ）

## 演出

### 全画面砂嵐 + ウィンドウ揺れ
- クイズ表示時・ボス登場時の両方に適用
- SVG feTurbulence による静的ノイズ（CSSアニメーション steps() で1.2秒）
- ウィンドウが揺れながら出現

### ボス登場アニメーション（ランダム）
- `is-glitch`：揺れながら出現
- `is-enter-flip`：rotateYで高速スピン（2回転）してから止まる
- 毎回ランダムで切り替わる

### システムメッセージ
- 俺レベ風「［ システム ］」形式
- タイプライター表示（1文字ずつ）、タップでスキップ
- プレイヤー名は認証したnote IDを使用

## 実装上の工夫・悩んだポイント

### ランク導出をボス討伐から行う設計
当初ワイ数でランクを判定していたが、「ボスを倒さずにワイを溜めるとランクが上がってしまう」問題が発生。解決策として：
- `kitacoreRankOf(creatorId)` → 討伐済みボスから最高ランクを導出
- `kitacoreWaiRankOf(totalWai)` → ボス出現トリガー判定にのみ使用
の2関数に分離。

### ボス出現の順序制御
「2000ワイに達したら途中のボスを飛ばしてASHEN REAPERが出る」バグへの対処。
`showPostBoss()` 内で「idx番目のボスを出す前に0〜idx-1が全員討伐済みか確認」するガードを追加。

### ✨ボタンのレースコンディション
初回ロード時、`kitacoreQuizzes`がnullの状態で`renderArticles`が走り✨ボタンが出なかった。
`loadKitacoreQuizzes().then(renderRoute)` で解決。

### 認証フローの2段階化
最初は認証成功→0.8秒後に自動でモード発動していたが、アイコンが一瞬しか見えないという問題。
1回目「認証」でプレビュー表示、ボタンが「決定」に変わり、2回目で確定という2段階フローに変更。

### デバッグボタンの制御
`?debug=1` クエリパラメータかつ記事取得済みのときのみ表示。本番環境でKITAさんに確認してもらう際も、通常URLでは非表示になる。

## 技術スタック

- Vanilla JS / IIFE構造 / ビルド不要
- 状態は localStorage（`yomiasa:v0`）
- note APIはCORSプロキシ（Cloudflare Workers）経由
- Service Workerでオフラインキャッシュ（バージョンbumpでキャッシュ無効化）
- SVGアニメーション（feTurbulence / CSS steps()）
- ボス画像：WebP形式、アプリ同梱
